Built for precision

Everything you need to catch visual regressions.

Side-by-side diffs

Each diff region is cropped and placed side by side. See exactly what changed at a glance.

Pixel-level accuracy

Configurable threshold from exact match to lenient. Catches subpixel rendering differences.

CLI + programmatic

Run from the terminal or import as a library. JSON and base64 output modes for CI pipelines.

Multi-URL support

Pass up to 100 URLs. Every unique pair is compared with concurrency control to keep CPUs happy.

info.txt report

Every run writes a plain-text summary. Readable by humans and AI agents, with image references.

Fully configurable

Viewport size, threshold, overlay opacity, outline color, region padding. Every visual detail is a flag.

Quick start

Up and running in seconds.

1 Compare two pages
$ px-diff https://staging.app https://prod.app
2 Check the results
$ cat .px-diff/info.txt

3 Compare multiple pages at once
$ px-diff https://a.com https://b.com https://c.com --concurrency 2
4 Use in CI with JSON output
$ px-diff url1 url2 --json --base64

Programmatic API

Import and use in your own scripts and test suites.

import { comparePair, defaults } from "px-diff";
const result = await comparePair(
"https://staging.app",
"https://prod.app",
{ ...defaults },
"staging-vs-prod",
);
result.diffPercent // 3.21
result.regions // [{ x1, y1, x2, y2 }, ...]
